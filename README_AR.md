# Heroku XMR Ultra Safe v7 - تطبيق afdaa

هذه النسخة مصممة لتقليل مشكلة:

```text
Process exited with status 137
XMRig خرج بالكود: -9
```

السبب غالبًا أن Heroku يقتل العملية عند استهلاك موارد/ذاكرة عالية. لذلك هذه النسخة تعمل بإعدادات منخفضة جدًا:

- `XMR_THREADS=1`
- `RANDOMX_MODE=light`
- `RESOURCE_PERCENT=25`
- تشغيل/إيقاف دوري لتقليل المتوسط العام للاستهلاك
- عند القتل بالكود `-9` يتم تخفيض الاستهلاك تلقائيًا وإعادة المحاولة بدون إسقاط worker نفسه

## أين أضع عنوان المحفظة؟

افتح ملف:

```bash
config.env
```

وغيّر هذا السطر:

```bash
XMR_WALLET="PUT_YOUR_XMR_RECEIVE_ADDRESS_HERE"
```

إلى عنوان الاستلام XMR الخاص بك:

```bash
XMR_WALLET="عنوان_XMR_الخاص_بك"
```

أو ضع عنوان XMR فقط داخل ملف:

```bash
wallet.txt
```

الأفضل استخدام `config.env`.

لا تضع كلمات الاسترداد أو المفتاح الخاص نهائيًا.

## طريقة الرفع من موقع Heroku أو GitHub

1. عدّل `config.env` وضع عنوان المحفظة.
2. ارفع الملفات إلى GitHub أو Heroku حسب طريقتك.
3. من لوحة Heroku افتح تطبيق `afdaa`.
4. من تبويب Resources اجعل:

```text
worker = 1
```

لا تجعل worker = 2 أو أكثر.

5. افتح Logs وانتظر ظهور:

```text
accepted
```

إذا ظهرت `accepted` فالتعدين يعمل.

## إذا ظهر XMRig خرج بالكود -9

هذا يعني أن Heroku قتل عملية XMRig. النسخة ستعيد المحاولة تلقائيًا بوضع أخف.

إذا تكرر كثيرًا، عدّل `config.env` إلى:

```bash
RESOURCE_PERCENT="10"
DUTY_WORK_SECONDS="3"
DUTY_SLEEP_SECONDS="30"
```

ثم أعد النشر.

## ملاحظات مهمة

- التعدين على Heroku قد يكون ضعيف الربح لأن الوضع الآمن يقلل الاستهلاك.
- لا يمكن ضمان عدم القتل 100% إذا كانت المنصة نفسها توقف العمليات كثيفة الموارد.
- هذه النسخة تجعل التطبيق لا يتحطم فورًا، وتخفف الذاكرة والـ CPU قدر الإمكان.
